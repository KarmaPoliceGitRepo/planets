# Literature Knowledge Base

A structured knowledge base distilled from the Google Drive folder **"0000 -03- LITERATURES"**
(645 files inventoried across 41 subject folders). Every entry is an original summary — no
copyrighted text is reproduced.

---

## What is here

| Artefact | Contents |
|---|---|
| `concepts/` | 203 canonical concept pages (one definition per term) |
| `notes/` | 671 document notes across 41 subject folders |
| `GLOSSARY.md` | Alphabetical index of every concept with its aliases |
| `INDEX.md` | Alphabetical index of every document note |
| `MANIFEST.md` | Per-folder coverage table (read / partial / skipped-duplicate / unreadable) |
| `_graphify/` | Durable knowledge digests by domain (ultrasonics, tribology, signal processing) |
| `_deep_read_queue.tsv` | Priority queue of documents awaiting deep-read upgrade |
| `_NOTE_TEMPLATE.md` | Canonical template for new document notes |
| `_DEEP_READ.md` | Step-by-step deep-read pipeline (PyMuPDF + figure vision) |

## How to navigate

**Finding a concept** — start at `GLOSSARY.md`. Each entry lists synonyms and links to the
single concept page in `concepts/`. Definitions live there; document notes reference the
concept rather than re-define it.

**Finding a document** — browse `INDEX.md` (documents sorted alphabetically, grouped by
folder) or open a subject folder directly under `notes/`. Subject folders map to the original
Drive sub-folders (e.g. `lubrication-film/`, `transducers-and-piezoelectricity/`,
`wear-measurement/`).

**Checking coverage** — `MANIFEST.md` shows the status of every Drive file:
`read` (324), `partial` (151), `skipped-duplicate` (88), `unreadable` (57), and smaller
counts for `collection`, `readable`, and un-processed items.

**Domain digests** — `_graphify/` holds three high-level synthesis files:
`ultrasonics-and-acoustics.md`, `tribology-manufacturing-instrumentation.md`, and
`methods-maths-signal-processing.md`. Read these for a fast orientation to a domain before
diving into individual notes.

## Core principles

**One canonical concept page.** If a term is used in many documents, it is defined once in
`concepts/<slug>.md`. Document notes list it under "Key concepts" and link to the concept
page instead of repeating the definition.

**Conflict resolution / quality hierarchy.** When sources disagree, the concept page notes
the conflict and favours peer-reviewed experimental papers over textbooks, and textbooks over
application notes or datasheets.

**No verbatim reproduction.** Every note is an original summary. Methods, equations, and
results are paraphrased or synthesised; long passages from source documents are never copied.

---

## Deep-read pipeline

Documents marked `partial` or `unreadable` in MANIFEST.md are queued for upgrade in
`_deep_read_queue.tsv` (priority-ordered, ultrasound/acoustics folders first).

The pipeline (`_DEEP_READ.md`) uses **PyMuPDF** for full text extraction page-by-page and
**figure vision** (render page to PNG → Read tool) for image-only or scanned PDFs. After a
deep read the note's Coverage field is updated to `full (deep read)` or
`visual read (scanned)`, folder sidecar TSVs are updated, and the queue row is marked `done`.

## How to extend

**Adding a note.** Copy `_NOTE_TEMPLATE.md`, fill in all fields, save to
`notes/<folder-slug>/<doc-slug>.md`. Update the folder's `_concepts.tsv` and `_manifest.tsv`
sidecars with any new concept rows and the file's new status.

**Adding a concept page.** Create `concepts/<slug>.md` using the pattern of existing pages
(title, aliases, definition, references). Add the entry to `GLOSSARY.md`.

**Regenerating indexes.** `GLOSSARY.md`, `INDEX.md`, and `MANIFEST.md` are generated by
script from the sidecars (`_concepts.tsv`, `_manifest.tsv`) in each folder and from the
concept pages. Re-run the generation script after bulk additions to keep them in sync.

**Updating domain digests.** After significant additions to a domain, update the relevant
file in `_graphify/` via `/graphify` to keep high-level knowledge current.
