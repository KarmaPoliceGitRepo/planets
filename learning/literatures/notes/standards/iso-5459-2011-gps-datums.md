## Overview
Second edition of ISO 5459, which defines terminology, rules, and graphical methodology for specifying datums and datum systems in technical product documentation (drawings, 3D models). Part of the ISO/GPS (Geometrical Product Specifications) matrix, covering chain links 1–3 for datums. Replaces the first edition (1981) with greatly expanded coverage of surface types and modern GPS concepts.

## Unique contribution
Extends datum specification from the three basic surface types (plane, cylinder, sphere) in the 1981 edition to all surface invariance classes (complex, prismatic, helical, cylindrical, revolute, planar, spherical). Formally defines the association criterion for establishing a datum from a real (non-ideal) surface feature, and introduces datum targets (points, lines, areas) and moveable datum targets. Introduces collection surfaces for common datums.

## Key concepts
- Datum feature (real surface used to establish a datum)
- Associated feature (ideal geometry fitted to datum feature)
- Datum (situation feature: point, line, plane derived from association)
- Primary / secondary / tertiary datums in a datum system
- Single datum, common datum, datum system
- Datum target (point, line segment, area subset of datum feature)
- Invariance classes (7 classes: complex, prismatic, helical, cylindrical, revolute, planar, spherical)
- Association criterion (objective function + constraints)
- Theoretically exact dimension (TED)
- Graphical symbols: datum feature indicator, datum identifier

## Methods / results / takeaways
The specification operator for a datum involves: (1) selecting the datum feature on the workpiece, (2) choosing the association criterion (default is minimum zone or minimum circumscribed, per Annex A), and (3) extracting the situation feature(s) from the associated ideal geometry. The default association criteria by surface type are given in normative Annex A. For a primary planar datum, the default is minimum distance association with the material constraint that the associated plane lies outside the material. Datum systems fix progressively fewer degrees of freedom: primary removes the most (e.g., 3 rotations for a plane), secondary removes orientation constraints, tertiary removes remaining location constraints. Examples in Annex C cover complex cases including spheres, cylinders-in-V-blocks, and assembly datums.
