# 2 · Stakeholder Needs (User-Suitable Statements)

> **SE step:** *Needs definition.* A **need** is written in the stakeholder's own
> plain language — *what they want and why* — **not** a technical solution.
> Good needs are the bridge between people and engineering. Each need gets an ID
> (`N-xx`) so we can trace it forward to a requirement later.
>
> **Format:** *"As a **\<stakeholder\>**, I need **\<capability\>** so that
> **\<benefit\>**."*

## 2.0 Collective needs (group level)

Each stakeholder **group** from [`01-stakeholders.md`](01-stakeholders.md) §1.3 holds
one **collective need**. The per-stakeholder needs below (`N-xx`) are *refinements*
of these parents — every detailed need rolls up to exactly one collective need.

| Collective need | Group | Refined by (detailed needs) |
|---|---|---|
| **CN-G1** Audience: find, play, follow relevant episodes — reliably, accessibly, on any platform/device. | G1 | N-10, N-11, N-12, N-13, N-21, N-25, N-26 |
| **CN-G2** Content originators: plan→record→edit→publish quality episodes at zero cost / low skill, with fair attribution. | G2 | N-01…N-09, N-22 |
| **CN-G3** Subjects & beneficiaries: be represented fairly; gain visibility/income/support from the exposure. | G3 | N-16, N-17, N-33 |

## 2.1 Host / Creator (S1)

- **N-01** As the host, I need to record **clear, listenable audio** at home so my message isn't lost to hiss and echo.
- **N-02** As the host, I need to also capture **video** (I have a camera) so I can publish on YouTube and social media, not just audio.
- **N-03** As the host, I need to **edit out mistakes** (ums, long pauses, fluffed lines) so episodes feel tight and professional.
- **N-04** As the host, I need the whole thing to be **free or near-free** so I can keep going without a budget.
- **N-05** As the host, I need steps **simple enough that a child could follow them** so I'm never blocked by technical complexity.
- **N-06** As the host, I need to **publish to where people already listen** (Spotify, Apple, YouTube) so I don't have to build an audience from scratch.
- **N-07** As the host, I need a **repeatable weekly routine** so the show is consistent and I don't reinvent the process each time.
- **N-08** As the host, I need to **keep my recordings safe** so I never lose work if my laptop breaks.
- **N-09** As the host, I need a **clear call-to-action** in each episode so I can grow a community and later promote the guide platform.

## 2.2 Listeners (S2, S3, S4)

- **N-10** As a diaspora listener, I need the show **on my normal podcast app and YouTube** so I can listen on my commute or watch at home.
- **N-11** As a listener with patchy internet, I need **reasonable file sizes** so episodes download without eating my data.
- **N-12** As a bilingual listener, I need content I can follow in **Nepali or English** so language is never a barrier.
- **N-13** As a busy listener, I need **show notes, chapters and links** so I can jump to what interests me and find sources.

## 2.3 Guests & Local guides (S5, S6)

- **N-14** As a remote guest, I need to join from **my phone or laptop wherever I am** so distance isn't a blocker.
- **N-15** As a guest in a village with weak signal, I need a way to **record my side locally** so my audio is clear even if the call drops.
- **N-16** As a local guide, I need the episode to **credit me and link to how to reach me** so I get real benefit (visibility, bookings).

## 2.4 Subjects & ethics (S7)

- **N-17** As someone whose story is told, I need to be **represented fairly and with consent** so I'm not misquoted or exploited.

## 2.5 Platforms, rights holders, regulators (S8, S10, S14)

- **N-18** As a platform, I need the show to **meet my technical & content rules** (file format, metadata, no banned content) so it can be listed.
- **N-19** As a rights holder, I need my music/images to **not be used without permission** so my rights aren't infringed.
- **N-20** As a regulator/culture, I need content to be **lawful and culturally respectful** so it can be safely published in/about Nepal.

## 2.6 Accessibility & helpers (S11, S13)

- **N-21** As a deaf or hard-of-hearing listener, I need **captions/transcripts** so I can access the content.
- **N-22** As a helper (sibling/parent), I need tools that are **obvious and low-effort** so I can pitch in without training.
- **N-23** As a parent of a minor creator, I need account sign-ups to be **age-appropriate and safe** so my child can participate responsibly.

## 2.7 Future sponsors (S12)

- **N-24** As a future sponsor, I need **basic audience numbers and a professional brand** so I can justify supporting the show.

## 2.8 Newly-registered stakeholders (S15–S22, S12b)

- **N-25** As a **discovery listener** (S15), I need the show **surfaced by recommendations and related-content**, with good titles/tags, so I find it without searching.
- **N-26** As an **offline/premium listener** (S16), I need **downloadable audio** so I can listen with no signal or in the background.
- **N-27** As an **advertiser / sponsor** (S17, S18), I need **verifiable listen/view counts and brand-safe content** so I can justify my spend on host-read or placed ads.
- **N-29** As a **playback device** (S19), I need **standard formats** (MP3, MP4/H.264/AAC) so the show plays on any phone, speaker, or TV.
- **N-30** As an **in-field creator on weak signal** (S20), I need **low-bandwidth upload and local capture** so connectivity never blocks recording or publishing.
- **N-31** As **platform Trust & Safety** (S21), I need each episode to **pass community-guideline and age checks** so it is not demonetised or removed.
- **N-32** As an **interfacing social platform** (S22), I need **clean vertical clips and embeddable previews** so shared links render and survive re-compression.
- **N-33** As the **promoted venture** (S12b), I need every episode to **carry a call-to-action and community link** so listeners can act on it.

## 2.9 Needs → priority (MoSCoW)

To keep a beginner from drowning, we prioritise. **Must** = v1 launch.

| Priority | Needs |
|---|---|
| **Must have** | N-01, N-03, N-04, N-05, N-06, N-07, N-08, N-18, N-19, N-20 |
| **Should have** | N-02, N-09, N-10, N-13, N-21, N-29, N-33 |
| **Could have** | N-11, N-12, N-14, N-15, N-16, N-22, N-24, N-25, N-26, N-27, N-30, N-31, N-32 |
| **Won't (yet)** | N-17 stays a *standing ethical rule*; N-23 handled by a guardian; full bilingual dubbing deferred |

> 🧭 **Reading the priorities:** Launch the **audio** show well first (Musts).
> Add **video, community, accessibility** next (Shoulds). Layer in **remote
> guests, bilingual, sponsors** as you grow (Coulds).
